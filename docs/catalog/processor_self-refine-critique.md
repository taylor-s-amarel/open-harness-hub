# Self-Refine critique loop

*processor* · `processor/self-refine-critique` · v0.1.0 · beta

Critique-and-revise loop (Madaan et al. 2023). The same model first
drafts an answer, then critiques its own draft against a rubric,
then revises. Iterates up to `max_iterations`. Useful when a single
pass produces verbose or weakly-grounded output.
