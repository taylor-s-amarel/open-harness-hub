# Diamond Model attribution (CTI threat-actor analysis)

*pattern* · `pattern/diamond-model-attribution` · v0.1.0 · stable

Classical CTI threat-actor attribution framework. An incident is
analyzed across 4 vertices: Adversary (who) + Capability (how) +
Infrastructure (with what) + Victim (against whom). Each vertex
is corroborated independently; attribution requires ≥2 vertices
with high-confidence evidence.

Counters the "single-IOC attribution" failure mode where one
hash or one domain leads to a published attribution claim.

| axis | value |
|---|---|
| industry | security, threat_intelligence |
| capability | reasoning, verification, evaluation |
| modality | text |
